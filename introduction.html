Introduction¶

Lava is a framework to build computational graphs for event-based, compute-in-memory, asynchronous, massively parallel and heterogenous neuromorphic hardware (such as Intel Corporation’s Neuromorphic Research Chip Loihi).

It consists of 4 main components:

  • Process - generic abstraction within the Lava framework for building reusable functional units of an algorithm or application.

  • Executable - While the process is an abstraction of reusable functional components, in order to execute the process on a given backend (CPU, GPU, or Neuromorphic Hardware), you must define a backend specific executable.

  • Variable - used to represent the state of a process as well as define its input/output interface such that processes may connect with each other.

  • Compiler - each supported backend requires a compiler. A given compiler consumes a process hierarchy, compiles them and produces an executor. When compiled, the associated executables utilize the executable variables (compiled variables) to carry out the functionality defined in the process on the chosen backend.
