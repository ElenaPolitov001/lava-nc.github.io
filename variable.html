Variable¶

class lava.core.generic.variable.AbstractExecVar(executable: lava.core.generic.executable.AbstractProcessExecutable)¶

Abstract base class for executor variables that implement the value of a Var for a specific backend.

  • For TF, we can simply subclass from ExecVar and tf.Variable.

  • For Nx, the ExecVar needs to hold the memory address within a neuro core

where the values of the variable can be found. There might be different variants to store the memory mapping in the most efficient way (e.g. explicit mapping, range-based-mapping, etc)

get_val and set_val must then interactively read/write the neuro core at that address.

A variable represents a node in a computational graph and provides a universal interface to the implementation of the variable’s value that may differ by backend.

Each variable must have a value, dtype and shape and is associated with a unique Process instance. Variables of different processes can be assigned/connected to each other in order to interconnect the respective processes.

The value of a variable is obtained via the ExecVar which provides a universal read/write interface regardless of the backend that implements the value.
